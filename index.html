Two payment rails

Lightning · L402

The default. On the first unauthenticated request, the server returns HTTP 402 with a BOLT11 invoice and an L402 token. Pay the invoice with any Lightning wallet — Phoenix, Zeus, Wallet of Satoshi, Alby, LNbits — then retry the same request with:

Authorization: L402 <token>:<preimage>

Verification is stateless — the server SHA-256s your preimage and checks the token HMAC. No sessions, no accounts, nothing durable for an attacker to steal.

Fiat credits · Bearer

If you'd rather pay by card once and forget it, buy a credit pack at api.loopxxi.com/ai-credits ($10 / $25 / $50 → sats at the live BTC price). You get a key of the form loop_…. Send:

Authorization: Bearer loop_<key>

Same per-call sats price, deducted from your prepaid balance. Loop Gateway does the atomic debit against the sats ledger.

60-second demo — pay a tool call by hand

# 1. Challenge — HTTP 402 with a real BOLT11 + token
curl -i https://mcp.loopxxi.com/l402/btc_price

# 2. Pay the payment_request in any Lightning wallet → get a 64-hex preimage

# 3. Retry with the L402 header
curl -i https://mcp.loopxxi.com/l402/btc_price \
  -H "Authorization: L402 <token>:<preimage>"

Prefer to click? Open the L402 Playground and watch your own 10 sats turn into an HTTP 200 response. Prefer to import a library? See the zero-dependency Python client and TypeScript client — MIT, ~150 lines each.

Wiring loop-mcp into a coding agent

loop-mcp is a remote HTTPS MCP server — nothing to install locally. Point any MCP-aware client at the endpoint:

{
  "mcpServers": {
    "loop-mcp": {
      "url": "https://mcp.loopxxi.com/mcp",
      "transport": "streamable-http"
    }
  }
}

Works with Cline, Roo Code, Cursor, Windsurf, Claude Desktop — any client that speaks streamable-HTTP MCP. The repo ships an llms-install.md that walks a coding agent through the exact configuration.
